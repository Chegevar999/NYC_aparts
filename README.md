## 1. Описание проекта

Проект выполнен по дисциплине **«Визуализация данных»**.

Тема проекта — **визуализация и анализ рынка недвижимости** на основе характеристик жилых домов из датасета **Ames Housing test dataset**.

В рамках проекта выполнен полный цикл аналитической работы:

- загрузка данных;
- первичный анализ данных;
- очистка и подготовка датасета;
- математико-статистический анализ;
- построение статических и интерактивных визуализаций;
- создание интерактивного Streamlit-дашборда;
- реализация ML-модели;
- визуальная интерпретация результатов;
- оформление GitHub-репозитория.

Так как в файле `test.csv` отсутствует целевая переменная `SalePrice`, задача машинного обучения реализована не как прогнозирование цены, а как **кластеризация объектов недвижимости методом KMeans**.

Кластеризация позволяет разделить дома на группы по похожим характеристикам: площади, качеству, году постройки, гаражу, подвалу и другим признакам.

### Основные шаги очистки

1. Удаление дубликатов.
2. Обработка пропущенных значений.
3. Заполнение числовых признаков медианой.
4. Заполнение категориальных признаков модой.
5. Приведение типов данных.
6. Обработка выбросов методом IQR.
7. Сохранение очищенного датасета.

### Обработка пропущенных значений

Пропущенные значения были обработаны следующим образом:

- для числовых признаков использовалась **медиана**;
- для категориальных признаков использовалась **мода**.

Такой подход выбран потому, что медиана устойчива к выбросам и лучше подходит для данных недвижимости, где значения площади могут сильно отличаться.

Мода подходит для категориальных признаков, так как позволяет заполнить пропуски наиболее часто встречающимся значением.

### Обработка выбросов

Для обработки выбросов использовался метод **IQR**.

IQR позволяет определить нижнюю и верхнюю границы допустимых значений:

```text
IQR = Q3 - Q1
Нижняя граница = Q1 - 1.5 * IQR
Верхняя граница = Q3 + 1.5 * IQR
```

Значения, выходящие за эти границы, ограничивались нижней или верхней границей.

### Почему выбросы не удалялись полностью

В недвижимости большие дома или большие участки могут быть реальными объектами, а не ошибками. Поэтому выбросы были не удалены полностью, а ограничены методом IQR.

Это позволяет сохранить больше данных и одновременно уменьшить влияние слишком экстремальных значений на анализ и ML-модель.

Так как в датасете отсутствует столбец `SalePrice`, задача прогнозирования цены невозможна без файла `train.csv`.

**Кластеризация**

### Использованная модель

**KMeans**

### Почему выбрана KMeans

Модель KMeans подходит для задачи группировки объектов, когда нет целевой переменной.

В данном проекте KMeans используется для разделения домов на группы по похожим характеристикам.

```text
TotRmsAbvGrd
```

Эти признаки выбраны потому, что они описывают основные характеристики объекта недвижимости:

- размер участка;
- качество дома;
- состояние дома;
- год постройки;
- жилую площадь;
- количество ванных;
- гараж;
- подвал;
- количество комнат.

### Подготовка данных для ML

Перед обучением модели были выполнены следующие действия:

1. Выбор числовых признаков.
2. Заполнение пропусков.
3. Масштабирование данных.
4. Обучение модели KMeans.
5. Добавление номера кластера в датасет.
6. Визуализация результатов через PCA.

### Масштабирование признаков

Для KMeans обязательно масштабирование, потому что модель использует расстояния между объектами.

Если признаки имеют разный масштаб, например `LotArea` измеряется тысячами, а `OverallQual` от 1 до 10, модель может неправильно учитывать важность признаков.

Поэтому использовался:

```text
StandardScaler
```

### Метрики модели

Для оценки модели использовались:

| Метрика | Значение |
|---|---|
| Inertia | показывает компактность кластеров |
| Silhouette Score | показывает качество разделения объектов |

### Inertia

Inertia показывает сумму расстояний объектов до центров своих кластеров.

Чем меньше Inertia, тем компактнее кластеры.

Но слишком маленькая Inertia может означать слишком большое количество кластеров, поэтому дополнительно используется Elbow Method.

### Silhouette Score

Silhouette Score показывает, насколько хорошо объекты разделены между кластерами.

Значение ближе к 1 означает хорошее разделение, значение около 0 означает пересечение кластеров, отрицательное значение означает возможное неправильное распределение объектов.

## 14. Визуализация и интерпретация результатов ML-модели

Результаты ML-модели представлены визуально.

### PCA 2D scatter plot

Так как модель использует несколько признаков, визуализировать их напрямую сложно.

Для этого использован метод PCA, который позволяет представить многомерные данные в двумерном пространстве.

PCA-график показывает, как объекты распределяются по кластерам.

### Elbow Method

Elbow Method использован для анализа выбора количества кластеров.

На графике отображается изменение Inertia при увеличении числа кластеров.

Оптимальное количество кластеров выбирается в точке, где снижение Inertia начинает замедляться.

### Интерпретация кластеров

Кластеры можно интерпретировать как группы домов со схожими характеристиками.

Примерная интерпретация:

| Кластер | Возможная интерпретация |
|---|---|
| Кластер 0 | дома со средними характеристиками |
| Кластер 1 | более крупные дома с большой жилой площадью |
| Кластер 2 | старые или менее качественные объекты |
| Кластер 3 | дома с более высоким качеством и большей площадью гаража |

Точная интерпретация зависит от средних значений признаков внутри каждого кластера.

### Ограничения ML-модели

KMeans не предсказывает цену недвижимости.

Модель только группирует объекты по похожим характеристикам.

Для полноценной регрессионной модели прогнозирования цены необходимо добавить файл `train.csv`, где есть столбец `SalePrice`.

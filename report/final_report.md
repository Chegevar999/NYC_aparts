## 2. Название проекта
**Визуализация и анализ рынка недвижимости на основе характеристик жилых домов**

## 3. Цель проекта
Цель проекта — провести полный цикл анализа данных недвижимости: EDA, очистку, математико-статистический анализ, визуализацию, интерактивный дашборд и ML-модель.

## 4. Описание выбранного датасета
Используется датасет Ames Housing test dataset. Он содержит 1459 объектов и 80 признаков. Данные описывают характеристики жилых домов: площадь участка, жилую площадь, район, качество, год постройки, гараж, подвал и другие параметры.

Важное ограничение: в `test.csv` отсутствует столбец `SalePrice`, поэтому прогнозирование цены невозможно без `train.csv`. Для соблюдения ML-блока была выбрана задача кластеризации объектов недвижимости.

## 5. Словарь признаков
| Признак | Тип | Описание | Использование |
|---|---|---|---|
| `GrLivArea` | числовой | жилая площадь дома | KPI, гистограмма, ML |
| `LotArea` | числовой | площадь участка | ML, статистика |
| `OverallQual` | числовой | общая оценка качества | сравнение, ML |
| `OverallCond` | числовой | общее состояние дома | ML |
| `YearBuilt` | числовой/временной | год постройки | фильтр, анализ |
| `GarageCars` | числовой | вместимость гаража | ML |
| `GarageArea` | числовой | площадь гаража | scatter plot, ML |
| `Neighborhood` | категориальный | район | фильтр, bar chart |
| `TotalBsmtSF` | числовой | площадь подвала | ML |

## 6. Первичный анализ данных / EDA
В ходе EDA были выполнены следующие действия:
- просмотр первых строк данных;
- определение размерности датасета;
- анализ типов признаков;
- выделение числовых и категориальных признаков;
- проверка пропусков;
- проверка дубликатов;
- расчёт описательной статистики;
- первичное выявление выбросов.

## 7. Очистка и подготовка данных
Очистка включала:
- удаление дубликатов;
- заполнение пропущенных числовых значений медианой;
- заполнение пропущенных категориальных значений модой;
- обработку выбросов методом IQR с ограничением значений нижней и верхней границей;
- сохранение очищенного датасета в `data/processed/clean_real_estate_with_clusters.csv`.

Такой подход выбран потому, что медиана устойчива к выбросам, а мода подходит для категориальных признаков.

## 8. Математико-статистический анализ
Для числовых признаков были рассчитаны:
- среднее;
- медиана;
- минимум и максимум;
- стандартное отклонение;
- дисперсия;
- квартили;
- корреляционная матрица.

Корреляционный анализ помогает определить, какие признаки имеют схожее поведение и могут быть полезны для ML-модели.

## 9. Основные визуализации
В проекте построены обязательные визуализации:
1. График пропущенных значений.
2. Гистограмма жилой площади.
3. Столбчатая диаграмма средней площади по районам.
4. Boxplot жилой площади по качеству дома.
5. Scatter plot площади гаража и жилой площади.
6. Корреляционная матрица.
7. PCA-визуализация кластеров.
8. Elbow method для KMeans.

## 10. Описание интерактивного дашборда
Дашборд реализован в Streamlit. Он содержит:

- описание датасета;
- KPI-карточки;
- фильтр по району;
- фильтр по году постройки;
- фильтр по жилой площади;
- фильтр по кластеру;
- Top-N фильтр районов;
- интерактивные графики Plotly;

## 11. ML-модель и визуальная интерпретация результатов
Выбрана модель KMeans, так как в датасете нет целевой переменной `SalePrice`. Кластеризация позволяет сгруппировать дома по похожим характеристикам.

Использованные признаки:
`LotArea`, `OverallQual`, `OverallCond`, `YearBuilt`, `GrLivArea`, `FullBath`, `GarageCars`, `GarageArea`, `TotalBsmtSF`, `TotRmsAbvGrd`.

Метрики:
- Inertia показывает компактность кластеров.
- Silhouette Score показывает качество разделения объектов на группы.

Визуальная интерпретация выполнена через PCA 2D scatter plot и elbow method.

## 12. Аналитические сценарии
1. Сравнение районов по средней жилой площади.
2. Анализ влияния качества дома на площадь.
3. Поиск групп похожих объектов через кластеризацию.
4. Анализ объектов по году постройки и площади.
5. Фильтрация данных для изучения отдельных сегментов недвижимости.

## 13. Основные выводы
- Датасет подходит для визуального анализа недвижимости.
- Наиболее полезные признаки для анализа: жилая площадь, площадь участка, качество, год постройки, район и гараж.
- Районы отличаются по характеристикам объектов.
- Кластеризация позволяет выделить группы недвижимости с похожими параметрами.
- Для полноценного прогнозирования цены нужен файл `train.csv` со столбцом `SalePrice`.

## 14. Ограничения данных и модели
- В данных отсутствует целевая переменная цены.
- KMeans чувствителен к масштабу признаков, поэтому использовалась стандартизация.
- Кластеризация не предсказывает цену, а только группирует объекты.
- Для улучшения проекта можно добавить `train.csv` и построить регрессионную модель цены.
